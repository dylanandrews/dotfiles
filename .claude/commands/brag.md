# Add to Brag Doc

Takes the content provided and adds it as a bullet point to the top of the brag document under this week's date heading, with today's specific date included in the bullet point. Each entry is tagged with the most relevant leveling expectation.

## Instructions

1. Get the current date in format "Week of YYYY-MM-DD" (using Monday of current week)
2. Get today's specific date in format "YYYY-MM-DD"
3. Read the existing brag doc at `/Users/dylanandrews/Dropbox/betterup_obsidian_vault/BetterUp/career/brag_doc.md`
4. Read the leveling expectations at `/Users/dylanandrews/Dropbox/betterup_obsidian_vault/BetterUp/career/leveling_expectations.md`
5. Check if there's already a heading for this week at the top
6. If the heading exists, add the new bullet point under it with today's date
7. If the heading doesn't exist, create a new bold heading for this week at the very top
8. Analyze the content against the leveling expectations and select the 1-2 strongest matches
9. Append the leveling tag(s) to the end of the bullet point using the format: `→ M1 Theme Name` or `→ P5 Theme Name`
10. Add the completed bullet point (content + leveling tags) under the appropriate heading
11. Preserve all existing content below

## Leveling Tag Format

Append to the end of each bullet point:
```
→ M1 Horizontal Leadership
```
or for two tags:
```
→ M1 Future-Ready Leadership + P5 Strategic Impact
```

## Tagging Guidelines

- **Prioritize M1 tags.** When work demonstrates both IC and management behaviors, lead with M1.
- Pick the **strongest, most specific match** — not every entry needs two tags
- **Match the specific behavior**, not just the theme name
- When in doubt between P5 and M1, prefer M1
- Skip the tag only if the entry is purely informational with no clear leveling signal
- Keep the tag brief — just the level and theme name, no quotes needed

## Document Format

```
**Week of YYYY-MM-DD**
* 2026-04-08: [brag item content] → M1 Horizontal Leadership
* 2026-04-06: [existing items...]

**Week of YYYY-MM-DD**
* 2026-04-01: [previous week items...]
```

## Usage

`/brag Successfully implemented the Cortex Gold status fix and got great feedback from the team`

This will add something like: `* 2026-04-08: Successfully implemented the Cortex Gold status fix and got great feedback from the team. → P4 Problem Solving`
